Уважаемый кандидат, поздравляем тебя! Если ты читаешь эту страничку, значит ты уже прошёл первичный отбор и осталось совсем чуть-чуть, чтобы присоедениться к нам.

Эта универсальная страничка для всех кандидатов, которая поможет подготовиться к собеседованию и выполнить тестовое задание.

Требования

  1. Можно без опыта в нагрузочном тестировании
  2. Обязательный опыт работы с консолью Linux. Уметь определять состяние системы - характеристики железа, общее здоровьё системы и отдельных её частей.
  3. Знать хотя бы один из перечисленных языков программирования на хорошем уровне - C, C++, C#, Java, Python, Go, JavaScript
  4. Знать популярные сетевые протоколы. Отлично знать HTTP/HTTPS.
  5. Хорошо знать HTML, уметь читать простой JavaScript код. Форматы JSON, XML.
  6. Иметь теоретические знания о нагрузочном тестировании. Какие основные метрики используются для оценки производительности системы.
  7. Знать SQL
  8. Знать Docker
  9. Технический английский язык

Тестовое задание

Написать скрипт, который успешно проходит тест, расположенный по адресу: http://test.youplace.net.

Скрипт должен быть написан на LoadRunner. Для этого необходимо скачать и установить бесплатную пробную версию с официального сайта.

Если вы никогда ранее не использовали этот инструмент, то вам сперва необходимо ознакомиться со справочной информацией и примерами кода.

Задание считается успешно выполненным, если после нескольких прогонах скрипта прямо из самой среды разработки в однопользовательском режиме не возникает ни одной ошибки, в том числе HTTP.

Не нужно запускать многопользовательский тест, ваш IP адрес может быть заблокирован сервером автоматически.

Тестовое задание должно быть полностью выполнено к первому собеседованию. Жёстких сроков по выполнению тестового задания нет, но следите, чтобы вам хватило бесплатного тестового периода на Loadrunner. Будьте готовы на собеседовании объяснить любой участок вашего кода.

Скрипт должен быть помещён на любой публичный GIT репозиторий. Тестовое задание присылается на проверку виде URL ссылки на GIT репозиторий. Скрипты, присланные по почте или выложенные на файлообменники, не рассматриваются.

Желаем удачи!
